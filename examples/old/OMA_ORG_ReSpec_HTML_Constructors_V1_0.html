Index

The Index or Table of Contents is constituted by three different sections:

Table of Contents
Constructed automatically, it contains the list of all the sections in the document.
Table of Figures
Constructed automatically, it contains a list of all the figures inserted in the document.
Table of Tables
Constructed automatically, it contains a list of all the tables inserted in the document.

Table of Contents

The code for defining the Table of Contents is:

<section id="toc"></section>

The guidelines for using the Table of Contents are:

  • Table of Contents (Index) is constructed automatically as long as the <section id="toc"></section> is defined at the beginning of the <body> of the document.

  • If <id="toc"> is not listed as part of the <section>, then the Table of Contents will not be displayed.

Table of Figures

The code for defining the Table of Figures is:

<section id="tof"></section>

The guidelines for using the Table of Figures are:

  • Table of Figures is constructed automatically as long as the <section id="tof"></section> is defined at the beginning of the <body> of the document.

  • If <id="tof"> is not listed as part of the <section>, then the Table of Figures will not be displayed.

The guidelines for using the Table of Tables are:

Following is a link to an example of how the Informative section is displayed:

See Introduction section as an example.

The code for this example is:

The guidelines for defining the Informative sections are:

  • <section class="informative"> element defines the beginning and <section> defines the end of a new informative section.

  • class ="informative" is used by ReSpec to mark this section as INFORMATIVE. This means that the content in this section is not mandatory.

  • ReSpec will automatically add a statement such as: "This section is non-normative".

  • As a good practice, add the comment: <!-- start introduction --> to indicate the beginning of the section and then at the end of the section add: <!-- /end introduction -->.

Common Sections

Sections are used to create chapters inside of a document. Sections can also be nested, (sections inside of sections).

Following is a link to an example of how a Common section is displayed:

See example at the beginning of this section, Common Sections.

The code for this example is:

<!-- start section 1 --> <section> <h2>Section Title </h2> <p>Text goes here.</p> </section> <!-- /end section 1 --> <!-- start section 2 --> <section> <h2>Another Sub-section</h2> <p> Text goes here. </p> </section> <!-- /end section 2 -->

The guidelines for defining the Common sections are:

  • <section> and </section> elements define the beginning and end of a new section.

  • <h2> element defines the title of the section.

  • Please use ONLY <h2> titles.

  • Comments need to be added at the beginning: <!-- start section x --> and at the end of the section: <!-- /end section x -->.

  • A section element, <section>, MUST be followed by <h2>,a header type 2 element. Otherwise, the content of the section will not be displayed in the Index.

The code for this example is:

<table id="4x3"> <caption> 4 columns x 3 rows</caption> <thead> <tr> <td></td> <td>Column 2</td> <td>Column 3</td> <td>Column 4</td> </tr> </thead> <tbody> <tr> <td>Row 1</td> <td>Cell 1-2</td> <td>Cell 1-3</td> <td>Cell 1-4</td> </tr> <tr> <td>Row 2</td> <td>Cell 2-2</td> <td>Cell 2-3</td> <td>Cell 2-4</td> </tr> <tr> <td>Row 3</td> <td>Cell 3-2</td> <td>Cell 3-3</td> <td>Cell 3-4</td> </tr> </tbody> </table>

The guidelines for creating tables are:

  • <table> element represents data with more than one dimension, in the form of a table.

  • <caption> element contains the title of the table. ReSpec will automatically display it at the bottom of the <table>.

  • Also, ReSpecwill use the <caption> value to automatically populate the title in the Table of Figures.

  • <thead> element represents the column labels (headers) for the parent <table> element.

  • <tbody> element represents a block of rows that form the body of the parent element, the <table>.

  • <tr> element represents a row of cells in a <table> element.

  • <td> element represents a data cell in a <table> element.

  • Note that some HTML editors support the editing of tables in a what you see is what you get approach

Try tables.

Try table editor to create a table for the first time.

Unfortunately the following rules MUST be followed when using this table editor. Otherwise, the table will not display correctly.

  • Add <thead> and </thead> elements to nest the first pair of <tr> and </tr>. This will create a displayable table header.

  • Substitute in this first pair of <tr> and </tr> all the table data elements <th> and </th> for the standard <td> and </td> elements.

  • Add <tbody> and </tbody> elements to group all the table rows.

The code for this example is:

<table id="4x3"> <caption>Merging Cells</caption> <thead> <tr> <td></td> <td>Column 2</td> <td>Column 3</td> <td>Column 4</td> </tr> </thead> <tbody> <tr> <rowspan="2">Merge Rows</td> <!-- rowspan = "2"merges two cells going down two rows from here--> <td>Cell 1-2</td> <td>Cell 1-3</td> <td>Cell 1-4</td> </tr> <tr> <!-- rowspan = "2"merges two cells going down two rows --> <td>Cell 2-2</td> <td colspan ="2">Cell 2-3</td> <!-- colspan = 2 will merge two cells towards the right accross two columns--> <!-- colspan = 2 implies that cell 2-4 needs to be removed --> </tr> <tr> <td>Row 3</td> <td>Cell 3-2</td> <td>Cell 3-3</td> <td>Cell 3-4</td> </tr> </tbody> </table>

The guidelines for merging cells are:

  • <rowspan="x"> attribute is used to span a row from the cell where it is defined to a number of rows (down) indicated by "x".

  • <colspan="x"> attribute is used to span a column from the cell where it is defined to a number of columns (to the right) defined by "x".

to Definitions

Once the definition has been inserted in the Definitions section, it can be used anywhere within the document.

Following is an example of how a link to definitions is displayed:

It is then easy to link to that term from any where within the document. You can even link when for some reason the terms are plural.

The guidelines for using linking to definitions are:

  • Once the term is defined in the Definitions section, it is possible to link back to it from anywhere within the document by writing the term to define between the <a> constructor; e.g.: <a>term</a>.

  • For further reading related to the <a> element: http://www.w3.org/wiki/HTML/Elements/aS

to Sections and Subsections

This section describes an easy way to create a link, from anywhere in the document, to any of the existing sections and subsections of the document.

Following is an example of how to links to sections and subsections are displayed:

This is a link to the section called Linking.

This other link points to the first subsection of the Linking section to Definitions.

The code for this example is:

<p>This is a link to the section called <a href="#linking">Linking</a>.</p> <p>This other link points to the first subsection of the <strong>Linking</strong> section <a href="#to-definitions">to Definitions.</a></p>

The guidelines for linking to sections and subsection are:

  • ReSpec marks automatically every section with its corresponding section id, e.g.: <section id="section_title">. Therefore, the user doesn't need to define a section id. ReSepc does it by default.

  • The user only needs to create the anchor <a> at the point where he/she wants to link to the section or subsection. This link is formulated as e.g.: <a href="#section_title">Section_Title</a>

  • Ensure that you use the SAME title as in the section or subsection. If the title has blank spaces you need to incorporate these blank spaces in the title as "-". E.g.: title = "to Definitions" the equivalent id="#to-definitions".

  • It is possible to use lower case when constructing the id value; see above.
